1

Créer la base de données sur Neon

PostgreSQL serverless gratuit — aucune carte bancaire requise

A. Créer votre compte et votre base

  1. Aller sur console.neon.tech → cliquer "Sign Up" (connexion avec GitHub recommandée)
  2. Cliquer "New Project"
  3. Nom du projet : alumnipath
  4. Région : choisir AWS EU West 2 (London) — la plus proche du Cameroun
  5. Nom de la base de données : alumnipath
  6. Cliquer "Create Project" ✓

B. Récupérer vos chaînes de connexion

  1. Dans le dashboard Neon → cliquer sur votre projet alumnipath
  2. Section "Connection Details" (en haut à droite)
  3. Dans le menu déroulant, sélectionner "Prisma"
  4. Vous verrez deux URLs — copiez-les séparément :
# URL 1 — DATABASE_URL (avec pooler, pour l'application)
DATABASE_URL="postgresql://alumnipath_owner:xxxxxxxx@ep-xxx-pooler.eu-west-2.aws.neon.tech/alumnipath?sslmode=require&pgbouncer=true&connect_timeout=15"

# URL 2 — DIRECT_URL (sans pooler, pour les migrations)
DIRECT_URL="postgresql://alumnipath_owner:xxxxxxxx@ep-xxx.eu-west-2.aws.neon.tech/alumnipath?sslmode=require"
La différence : DATABASE_URL contient -pooler dans le hostname et pgbouncer=true. DIRECT_URL n'a pas ces paramètres. Les deux sont nécessaires.
2

Pousser le code sur GitHub

Vercel déploie directement depuis un dépôt GitHub

  1. Aller sur github.com → "New repository"
  2. Nom du dépôt : alumnipath · Visibilité : Public → "Create repository"
  3. Sur votre machine, dans le dossier du projet :
git init
git add .
git commit -m "feat: AlumniPath initial commit"
git branch -M main
git remote add origin https://github.com/VOTRE-USERNAME/alumnipath.git
git push -u origin main
Remplacer VOTRE-USERNAME par votre nom d'utilisateur GitHub exact.
3

Déployer sur Vercel avec le bon nom

Configuration du projet pour obtenir l'URL exacte demandée

A. Importer le projet

  1. Aller sur vercel.com → "Add New Project"
  2. Connecter votre compte GitHub si ce n'est pas fait
  3. Sélectionner le dépôt alumnipath → "Import"
  4. Framework : Next.js (détecté automatiquement)

B. Configurer le nom du projet (IMPORTANT pour l'URL)

  1. Dans la page de configuration, cliquer sur "Project Name"
  2. Saisir exactement :
Alumnipath-NGABETKONGUEUN CYRILLE JUNIOR-23U2960
Vercel convertit les espaces en tirets dans l'URL. Le nom saisi ci-dessus donnera l'URL : alumnipath-ngabetkongueun-cyrille-junior-23u2960.vercel.app

C. Ajouter les variables d'environnement Neon

Dans la section "Environment Variables" avant de déployer :

VariableValeurDescription
DATABASE_URL Coller l'URL pooler Neon URL avec -pooler et pgbouncer=true
DIRECT_URL Coller l'URL directe Neon URL sans pooler — pour prisma db push
Cliquer sur "Add" pour chaque variable, puis mettre la valeur en "Sensitive" (cadenas) pour la sécurité.

D. Vérifier la commande de build

Dans "Build & Output Settings", la commande de build doit être :

prisma generate && prisma db push && next build

Cela créera automatiquement les tables dans Neon à chaque déploiement.

E. Lancer le déploiement

  1. Cliquer "Deploy" → patienter 2–3 minutes
  2. ✅ Build réussi : vous verrez une preview de l'app
  3. Cliquer "Continue to Dashboard"
  4. Votre URL de production sera affichée en haut
4

Ajouter des données de démonstration (optionnel)

80 réponses réalistes pour tester le tableau de bord

Sur votre machine locale, créer un fichier .env.local avec vos URLs Neon, puis :

# Installer les dépendances
npm install

# Pousser le schéma vers Neon (crée les tables)
npx prisma db push

# Remplir avec 80 réponses de démo
npm run db:seed

# Vérifier dans Neon Studio
npx prisma studio
